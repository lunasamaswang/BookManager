# BookManager v1.1.1 开发日志

## 版本目标

BookManager v1.1.1 的目标是在不改变数据库结构和原有业务能力的前提下，将基础图书管理窗口升级为更完整的“智能书房控制台”。本版本重点改善首页信息层级、BookBot 助手表现、中文文字显示和图书库表格的窗口适配能力。

## 版本背景

v1.0.1 已经完成 SQLite 双模式路径、Windows 打包和基础图书管理功能。v1.1.0 初步加入三栏布局和首页仪表盘，但界面仍有普通管理后台的感觉，部分中文文字、按钮和表格列在窗口模式下存在裁切。

因此，v1.1.1 没有继续扩展业务范围，而是集中处理视觉表达和可用性，让已有功能在常见 Windows 窗口尺寸下更加稳定。

## 涉及文件

- `app/ui/main_window.py`：三栏主窗口、导航和页面切换。
- `app/ui/dashboard_page.py`：首页仪表盘、快速找书、统计和书房状态。
- `app/ui/library_page.py`：图书列表、搜索、操作按钮和响应式列宽。
- `app/ui/bookbot_panel.py`：BookBot 助手栏和今日书签。
- `app/assets/styles.qss`：全局暖色书房视觉样式。
- `app/assets/study_banner.png`：首页欢迎横幅插画。
- `app/assets/bookbot_avatar.png`：BookBot 本地头像素材。

## 架构理解

项目保持简单分层：

- `main.py` 负责初始化数据库、加载 QSS 和启动主窗口。
- `MainWindow` 负责三栏结构、页面切换和页面间信号连接。
- `DashboardPage` 只展示统计、最近图书和快捷入口。
- `LibraryPage` 承接原有添加、编辑、删除、搜索和 Excel 导入功能。
- `BookBotPanel` 目前是静态助手界面，不连接 AI 服务。
- `book_service.py` 和 `import_service.py` 继续负责数据查询与导入。
- `database.py` 和 `path_utils.py` 继续负责数据库连接与运行模式路径。

本版本只调整 UI 层，没有把业务 SQL 或数据库路径判断写入界面代码。

## 主要修改内容

1. 主界面采用深色导航、中间内容区和右侧 BookBot 的三栏布局。
2. 默认首页增加欢迎横幅、藏书统计、快速找书、最近添加、任务和书房状态。
3. 使用本地 PNG 素材增强书房氛围，同时保留真实 Qt 控件和交互。
4. BookBot 增加头像、在线状态、对话气泡、占位按钮和今日书签。
5. 为中文标签、说明文字和按钮补充换行、最小尺寸和合理的尺寸策略。
6. 图书库根据表格 viewport 动态计算列宽，并为长文本保留 tooltip。
7. 操作列固定宽度，确保编辑和删除按钮在常见窗口尺寸下完整显示。
8. 极窄宽度下保留水平滚动条，避免内容无提示地被页面裁切。

## 遇到的问题与解决方法

### 中间内容区实际宽度不足

三栏布局中，左侧导航和右侧 BookBot 都占用固定宽度。窗口为 `1240×720` 时，图书表格 viewport 只有约 678px，无法容纳原先接近 1000px 的固定列宽。

解决方法是根据 viewport 实际宽度重新分配九列。操作列保持固定，书名优先获得弹性空间，作者、分类、位置和时间在较窄窗口中适当压缩。

### 操作列位于最右侧而被遮挡

原列宽总和超过 viewport 后，编辑和删除按钮只能滚动到最右侧才能看到。

解决方法是把操作列设为最高优先级，固定为足够容纳两个按钮的宽度，并在窗口或 viewport 尺寸变化时重新计算其他列。

### 中文文字和按钮被压缩

部分标签没有换行策略，按钮宽度又受布局挤压，导致中文只显示一部分。

解决方法是为说明文字启用 `setWordWrap(True)`，为关键按钮设置最小宽度，并通过精确的 objectName QSS 调整 padding，而不是全局缩小字体。

### 图片素材不能破坏交互

概念图不能直接作为整张界面背景，否则搜索、按钮和数据将失去真实交互能力。

解决方法是仅将图片用于欢迎横幅和 BookBot 头像，通过 `QLabel + QPixmap` 显示，并使用 `KeepAspectRatio` 保持比例。

## 本次学习收获

1. 桌面界面设计不能只看初始尺寸，还要根据真实可用区域处理伸缩。
2. `QHeaderView` 的 Stretch、Fixed 和 Interactive 策略需要结合业务优先级使用。
3. 对于数据表格，重要操作列应优先保证可见，长文本可以通过省略号和 tooltip 处理。
4. QSS 适合集中管理颜色和控件状态，尺寸行为仍需要在 Python 布局代码中控制。
5. UI 素材应服务于氛围表达，不能替代真实控件和业务交互。
6. 使用独立临时数据库回归测试，可以避免污染用户的正式数据。

## 比赛/论文表达

本项目采用 Python、PySide6 和 SQLite 构建本地个人图书管理系统。界面层使用三栏式信息架构，将导航、核心管理页面和智能助手占位区域分离；数据层继续通过统一服务接口访问 SQLite。

在响应式处理方面，系统根据表格 viewport 动态分配列宽，优先保证关键操作入口完整显示，并通过文本省略和 tooltip 兼顾有限空间与完整信息。这种设计体现了桌面应用中“业务优先级驱动布局”的思想。

在视觉实现方面，项目采用 Qt 原生控件、QSS 和本地图片素材混合方案，在保持交互真实性和打包稳定性的同时，形成温暖书房与轻科技结合的界面风格。
